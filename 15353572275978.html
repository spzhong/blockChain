工作量证明(POW)

2018/8/27 posted in  共识算法 comments

Proof of Work 工作量证明，也就是说你获得多少货币，取决于你挖矿贡献的有效工作，比如你电脑性能越好，分给你的矿就越多，即根据你的工作证明来执行活不的分配，大部分的虚拟货币，比如比特币，莱特币等都是基于POW模式的，算力越高，挖矿时间越长，获得货币就越多

实现的原理

不停的进行哈希计算，使其小于一个目标bits值的过程，称为“工作量证明”

每个区块中都有一个字段为bits的值

目标值bits:
4个字节，第一字节表示幂，后3个字节表示系数
例如：0x191a122f
bits = 0x1a122f 2 ^ (8 * ( 0x19 - 3 ))
矿工就要找出小于当前区块中的bits值


bits = 0x191a122f
hexBits = 0x1a122f ^ 8(0x19 -3 )

随着算力的增加，nonce随机数最大是232 ，如果算力每秒大于 232 HASH/秒，那么就会使用coinbase交易中的coinbase字段这里面预留了8个字节，加上nonce字段，就有了12个字节用于存储随机数，这个随机数最大支持296 HASH/秒，这是一个极大的数据，足够满足随机数的递增了

如何调整difficulty

为了调整挖矿产出时间，系统做了一个难度系数的调整，在每隔2016块（2周左右），系统会调整一下挖矿的难度，以防止后续过快或过慢的挖矿

难度值，只是一个浮点数，没有包含在区块数据中的，只是一个直观的体现

假设算力为t次Hash/秒

const int constant = 2^32

时间 = 难度 * constant / 算力
时间*算力 = N次HASH计算 = 难度 * constant

当算力恒定的情况下，难度越大需要的时间就越长
当时间恒定的情况下，难度越大需要的算力就越大

时间 = 难度 * constant / 算力
算力 = 难度 * constant / 时间
